# Proposal: Phase 4 — VOC 파이프라인 운영 전환

## Intent

수동 로컬 실행에 의존하던 VOC 파이프라인을 GCP 자동 운영 체계로 전환한다.
일간 분류·주간 리포트·대시보드를 단일 데이터 레이어(`voc_labelled.letters_posts`)
중심으로 정렬하고, 발행 전 안전장치를 도입한다.

### 현행 문제

1. **수동 실행 의존**: 매일/매주 Gayoon 노트북에서 `python scripts/...` 실행. 휴가/외출 시 운영 단절
2. **분류 비용 중복**: 주간 리포트가 원본을 다시 분류 (월 $84 추가) — 일간이 이미 분류한 결과를 재사용 안 함
3. **잘못된 리포트 발행 위험**: 적재 실패/잘못된 스크립트 선택 시 "데이터가 부족합니다" 같은 LLM 거부 답변이 그대로 발행됨 (2026-04-13 사고)
4. **부적절 인용구 노출**: 1차 프롬프트 가이드만으로는 욕설/인신공격성 인용구가 그대로 발행됨 (2026-04-13 "주갤 출신/가족까지 팔아먹지만" 사고)
5. **외부 공유 불가**: Streamlit 대시보드가 로컬 JSON 파일에 의존 → URL 공유 불가능

## 제안: 4-tier 자동 운영

```
[수집]                      [분류·저장]                  [발행]                  [조회]
us_plus_next  ──→  voc-daily Cloud Run Job  ──→  voc-weekly Cloud Run Job  ←─→  voc-dashboard
(외부 적재)         (매일 08:00 KST)              (매주 월 09:00 KST)              Cloud Run Service
                       ↓                                ↓                          (상시)
                   voc_labelled.letters_posts ──────────┘
                   (DATE 파티션, id 기준 dedup)
```

### 컴포넌트별 책임

| 컴포넌트 | 트리거 | 입력 | 출력 |
|---------|--------|------|------|
| `voc-daily` | Cloud Scheduler 매일 08:00 | 어제 KST 일자의 us_plus_next 원본 | voc_labelled 적재 + 부정 급증 Slack |
| `voc-weekly` | Cloud Scheduler 매주 월 09:00 | 지난 주 voc_labelled | Notion 페이지 + Slack 알림 + 엑셀 첨부 |
| `voc-dashboard` | 사용자 브라우저 | voc_labelled (최근 7/14/30일 / 커스텀) | Streamlit 웹 UI |
| `cluster_tech_issues` | Gayoon 수동 (분기 1회) | 로컬 voc_labelled export | 클러스터링 엑셀 (개발팀 분기 분석) |

### 안전장치 3중

1. **분류 단계** — 일간 파이프라인이 마스터별 부정 30%+ 감지 시 Slack DM (기존)
2. **주간 리포트 sanity check** (룰) — voc_labelled 비어있음/평일 0건/전주 대비 -50%+ 감지 시 발행 중단
3. **주간 리포트 인용구 검수** (LLM) — 욕설/인신공격/사기 단정 자동 제거 (`quote_reviewer.py`)

## Out of Scope (이번 단계 제외)

- 채널톡 KcELECTRA 자동 분류 — 일간에서 `--skip-channel`. 메모리/모델 검증 후 별도 phase
- 일간 완료 시 Slack 모니터링 메시지 (대시보드 URL + 부정 요약) — 별도 작업
- 일간 파이프라인 자체 실패 시 Slack 알림 — 별도 작업
- 대시보드 IAP 인증 — 초기 `--allow-unauthenticated`, 운영 시작 후 적용
- 분기 클러스터링 자동화 — 활용 빈도가 낮아 수동 유지

## 비용 영향

| 항목 | Before (수동) | After (운영) |
|------|--------------|------------|
| 일간 분류 | $0 (안 돌아감) | $90/월 (Bedrock Haiku) |
| 주간 분류 | $84/월 (수동, 원본 재분류) | $0 (voc_labelled 재사용) |
| Cloud Run 런타임 | $0 | <$2/월 |
| 합계 | ~$84/월 | **~$90/월** |

비용은 비슷하지만 일간 분류가 새로 추가되어 **매일 신선한 데이터 + 대시보드 + 부정 급증 알림** 가치 확보.

## 의사결정 기록

- **sanity check는 룰 기반 유지** (LLM 안 씀): 임계치 명확한 검사는 룰이 빠르고 결정적. LLM은 "리포트 본문 거부 답변 감지" 같은 자연어 의미 판단에만 적합 (현 단계 보류)
- **인용구 검수는 2레이어**: 1차 프롬프트 가이드(저렴) + 2차 LLM 독립 검수(안전망). 1차만으로는 뚫림이 확인됨
- **주간은 voc_labelled 읽기 전용**: 일간이 적재한 분류 결과를 재사용. 분류는 한 번만, 활용은 여러 번
- **개발팀 클러스터 뷰는 보류**: 분기 1회 수동 실행이 비용/효용 균형 좋음. 자동화는 활용 빈도 확인 후
- **GCP 리소스 생성은 사람이 직접**: Rocky가 `gcloud` 명령 임의 실행 안 함. 배포 스크립트는 작성, 실행은 Gayoon
